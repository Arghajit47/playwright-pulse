Changelog

Version 0.2.2

  • Added: Introduced 'System Information' widget to visualize details about the test execution environment (Host, OS, CPU, Memory, Node, etc.).
  • Enhanced Analytics: Improved accuracy in history trend calculations and added support for filtering trends by date range. Added workerIndex support for individual tests.
  • Bug Fixes: Resolved issues with inconsistent test suite durations and fixed layout glitches on smaller screens.
  • Improvements: Optimized report generation for large datasets and enhanced error handling. Added accurate browser name, version, and OS to test suites. Implemented lightweight HTML for email reports.

Version 0.2.1

  • Feature: Added history trends for the last 15 runs for test suites, suite duration, and individual tests.
  • Fixed: Corrected Project Name components in the test suite view.

Version 0.2.0

  • Major Update: Refined UI for static HTML reports.
  • Feature: Implemented History Trends for test suites, duration, and individual test executions.

Version 0.1.6

  • Fix: Addressed and fixed email report generation issues.

Version 0.1.5

  • Enhancement: Updated and fixed various styling issues in the report.

Version 0.1.4

  • Feature: Added the AI Analyzer tab and integration into the report.

Version 0.1.3

  • Feature: Introduced the capability to generate and send email-able reports.

Version 0.1.2

  • Improvement: Fixed browser filtering logic and added a Browser Tag to the Test Suite Card for easier identification.

Version 0.1.1

  • Performance: Added initial support for test sharding.
